ComObjType() [v1.0.91+]

Ermittelt Informationen über den Typ eines COM-Objekts.

Variablentyp := ComObjType(ComObject)
Name    := ComObjType(ComObject, "Name")
IID     := ComObjType(ComObject, "IID")

Parameter

ComObject Ein Wrapper-Objekt mit einem COM-Objekt oder typisierten Wert.
Variablentyp Ein Integer-Wert, der den Typ des Wertes kennzeichnet. Siehe unten.
Name Der Typenname einer Schnittstelle.
IID Eine globale eindeutige ID (GUID), die einen Schnittstellentyp repräsentiert.

Rückgabewert

Wenn der zweite Parameter weggelassen wird, dann wird der Variantentyp des Wertes vom Wrapper-Objekt zurückgegeben.

Wenn der zweite Parameter "Name" oder "IID" und das Wrapper-Objekt ein COM-Objekt enthält (Variantentyp VT_DISPATCH), dann wird der Typenname des COM-Objekts oder die Kennung der primären Schnittstelle zurückgegeben.

Wenn die Parameter ungültig sind oder ein Fehler auftritt, dann wird ein leerer String zurückgegeben.

Variantentyp-Konstanten

VT_EMPTY     =      0  ; Kein Wert
VT_NULL      =      1  ; Eine 0 im SQL-Stil
VT_I2        =      2  ; 16-Bit signed Int
VT_I4        =      3  ; 32-Bit signed Int
VT_R4        =      4  ; 32-Bit Fließkommazahl
VT_R8        =      5  ; 64-Bit Fließkommazahl
VT_CY        =      6  ; Währung
VT_DATE      =      7  ; Datum
VT_BSTR      =      8  ; COM-String (Unicode-String mit Längenpräfix)
VT_DISPATCH  =      9  ; COM-Objekt 
VT_ERROR     =    0xA  ; Fehlercode (32-Bit Integer)
VT_BOOL      =    0xB  ; boolesches Wahr (-1) oder Falsch (0)
VT_VARIANT   =    0xC  ; VARIANT (muss mit VT_ARRAY oder VT_BYREF kombiniert werden)
VT_UNKNOWN   =    0xD  ; IUnknown-Interface-Pointer
VT_DECIMAL   =    0xE  ; (nicht unterstützt)
VT_I1        =   0x10  ; 8-Bit signed Int
VT_UI1       =   0x11  ; 8-Bit unsigned Int
VT_UI2       =   0x12  ; 16-Bit unsigned Int
VT_UI4       =   0x13  ; 32-Bit unsigned Int
VT_I8        =   0x14  ; 64-Bit signed Int
VT_UI8       =   0x15  ; 64-Bit unsigned Int
VT_INT       =   0x16  ; Signed Maschinen-Int
VT_UINT      =   0x17  ; Unsigned Maschinen-Int
VT_RECORD    =   0x24  ; Benutzerdefinierter Typ
VT_ARRAY     = 0x2000  ; SAFEARRAY
VT_BYREF     = 0x4000  ; Pointer auf einem anderen Werttyp
/*
 VT_ARRAY und VT_BYREF werden mit einem anderen Wert kombiniert (verwendet bitweises OR),
 um den exakten Typ anzugeben. Zum Beispiel ermittelt 0x2003 ein SAFEARRAY mit 32-Bit signed Integers und 0x400C ein Pointer auf VARIANT.
*/

Allgemeine Bemerkungen

In den meisten Fällen werden Rückgabewerte von Methoden oder Eigenschaften von COM-Objekten in einen geeigneten Datentyp umgewandelt, der von AutoHotkey unterstützt wird. Nicht speziell behandelte Typen werden durch VariantChangeType zu Strings erzwungen. Wenn dabei ein Fehler auftritt oder der Variantentyp das VT_ARRAY- oder VT_BYREF-Flag enthält, dann wird stattdessen ein Objekt mit dem Wert und dessen Typ zurückgegeben.

Wenn ComObjType(x) (wobei x eine beliebige Variable ist) ein Integer zurückgibt, dann enthält x ein COM-Objekt-Wrapper.

Verwandte Befehle

ComObjValue, ComObjCreate, ComObjGet, ComObjActive

Beispiele

d := ComObjCreate("Scripting.Dictionary")
Variablentyp := ComObjType(d)          ; Immer 9 bei aufrufbare Objekte für Scripts.
Name    := ComObjType(d, "Name")  ; Nur bei aufrufbare Objekte für Scripts gültig.
IID     := ComObjType(d, "IID")   ; Wie oben.
MsgBox Variantentyp:`t%Variablentyp%`nTypenname:`t%Name%`nSchnittstelle-ID:`t%IID%
